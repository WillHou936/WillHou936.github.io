Post 命名规则

文件名

格式：<文章类型>-<知识点>-<标题>

文章类型：topic 或者 solution

知识点：

知识点名称 代码
算法基础 basicalg
排序 sort
位运算 bit
二分 bins
深搜 dfs
宽搜 bfs
贪心 greed
分治 DaC
剪枝 prun
动态规划 dp
背包 DP knapsack
区间 DP interval
树形 DP treedp
数据结构 ds
栈 stack
队列 queue
哈希 hash
二叉树 bintree
堆（优先级队列） heap
图 graph

如果同时属于多个知识点，就用 _ 连接它们。

能细则细，例如，有一个题目，知识点涉及到“栈”，那么知识点应该这样描述：ds_stack

标题：根据题目名字提炼。如果是 topic，则不需要该项。

当然，如果特殊情况，可以按需调整。

config - title

格式：<文章类型><标题>

config - categories

目前分为 “笔记”，“题解”，“测试”，“趣味” 四种，以后按需调整。

config - tags

与标题一一对应，但要写中文。其内容还包括语言名称，以便将来内容更加丰富时方便维护。
